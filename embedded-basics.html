嵌入式系统基础概念

什么是嵌入式系统？

嵌入式系统是专为特定应用而设计的计算机系统，通常集成在更大的系统中。在航天领域，嵌入式系统负责：

  • 飞行控制和导航
  • 数据采集和处理
  • 通信和遥测
  • 载荷管理
  • 故障检测和保护

嵌入式系统特点

实时性

必须在规定时间内响应和处理事件

低功耗

优化能耗，延长系统工作时间

可靠性

在恶劣环境下稳定运行

资源受限

内存、处理能力和存储空间有限

航天嵌入式系统架构

应用层

飞行控制算法、导航算法、载荷控制

操作系统层

实时操作系统（RTOS）、任务调度

驱动层

传感器驱动、执行器驱动、通信驱动

硬件层

微控制器、传感器、执行器、通信模块

核心代码示例

GPIO控制


// GPIO初始化
void MX_GPIO_Init(void) {
    GPIO_InitTypeDef GPIO_InitStruct = {0};
    __HAL_RCC_GPIOA_CLK_ENABLE();
    
    GPIO_InitStruct.Pin = GPIO_PIN_5;
    GPIO_InitStruct.Mode = GPIO_MODE_OUTPUT_PP;
    GPIO_InitStruct.Speed = GPIO_SPEED_FREQ_LOW;
    HAL_GPIO_Init(GPIOA, &GPIO_InitStruct);
}

// LED控制
void LED_Toggle(void) {
    HAL_GPIO_TogglePin(GPIOA, GPIO_PIN_5);
}

UART通信


UART_HandleTypeDef huart2;

void MX_USART2_UART_Init(void) {
    huart2.Instance = USART2;
    huart2.Init.BaudRate = 115200;
    huart2.Init.WordLength = UART_WORDLENGTH_8B;
    huart2.Init.StopBits = UART_STOPBITS_1;
    huart2.Init.Parity = UART_PARITY_NONE;
    HAL_UART_Init(&huart2);
}

void UART_SendString(char* str) {
    HAL_UART_Transmit(&huart2, (uint8_t*)str, strlen(str), 1000);
}

定时器应用


TIM_HandleTypeDef htim2;

void MX_TIM2_Init(void) {
    htim2.Instance = TIM2;
    htim2.Init.Prescaler = 8400-1;  // 10kHz
    htim2.Init.Period = 10000-1;    // 1Hz
    HAL_TIM_Base_Init(&htim2);
}

// 定时器中断回调
void HAL_TIM_PeriodElapsedCallback(TIM_HandleTypeDef *htim) {
    if(htim->Instance == TIM2) {
        LED_Toggle();  // 每秒闪烁一次
    }
}

常见问题

CubeMX高版本无法打开低版本工程

问题描述：

使用新版本STM32CubeMX打开旧版本创建的.ioc工程文件时，提示版本不兼容无法打开。

解决方案：

1
方法一：修改版本号

使用记事本打开.ioc文件，找到版本号行，修改为当前CubeMX版本号

ProjectManager.CubeMxVersion=6.15.0
2
方法二：软件更新

直接更新STM32CubeMX到最新版本，获得更好的兼容性

ST-LINK无法烧录程序

问题描述：

ST-LINK调试器连接开发板后，无法正常烧录程序到STM32芯片中。

解决方案：

1
检查硬件连接

确认ST-LINK与开发板连接正常，电源指示灯亮起

2
复位烧录法

长按开发板上的复位键，同时点击烧录，在烧录开始后松开复位键

3
更换调试器

如果问题持续存在，考虑更换ST-LINK调试器硬件

编译错误和调试技巧

常见编译问题：

1
头文件路径错误

检查Include Paths设置，确保所有必要的头文件路径已添加

2
函数未定义

确认函数声明和定义匹配，检查是否遗漏源文件添加到工程

3
内存溢出

检查Stack和Heap大小设置，优化代码减少内存占用
